We are building a mobile system that helps guide a bystander through CPR during a medical emergency using a phone camera and a voice assistant. The system is divided into three main components, each handled by a different person on the team. The goal is to keep responsibilities clearly separated so the system remains simple and modular.

The first component is the voice agent, which is responsible for interacting with the user through speech. This part listens to the user through the microphone, converts speech to text, interprets the response, and generates spoken instructions. The voice agent asks simple questions such as whether the person is responsive or breathing normally, and guides the user step by step through CPR instructions. The voice agent should ask one question at a time, wait for the answer, and keep responses short and clear so they are easy to follow in stressful situations.

The second component is the computer vision module, which uses the phone’s camera to observe what the rescuer is doing during CPR. This module detects the chest area of the patient, identifies whether the rescuer’s hands are positioned correctly, and estimates the rhythm of chest compressions. The computer vision system outputs simple signals such as whether hand placement is correct or incorrect and whether the compression rhythm is too slow, correct, or too fast. These signals are sent to the state manager so the voice agent can give real-time feedback.

The third component is the state manager, which acts as the central controller of the system. The state manager keeps track of the current stage of the CPR protocol and stores key information such as whether the patient is responsive, whether they are breathing normally, whether CPR has started, and what the current compression feedback is from the camera. The state manager receives answers from the voice agent and updates the system state accordingly. It also receives signals from the computer vision module and converts them into structured values such as hand placement status and compression rhythm status. Based on this information, the state manager determines the current step of the protocol and sends the relevant context back to the voice agent so it knows what to say next.

The overall system works as follows: the voice agent asks the user questions and sends the answers to the state manager. The computer vision module continuously analyzes the camera feed and sends CPR feedback signals to the state manager. The state manager combines both sources of information, updates the current protocol state, and provides context to the voice agent so it can generate the next instruction or question. This structure ensures that the conversation, the visual analysis, and the protocol logic remain clearly separated while working together in real time.
